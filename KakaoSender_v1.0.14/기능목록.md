# 카카오톡 자동 전송기 - 기능 목록 (v1.0.0)

## 1. 실행 및 환경 설정

| # | 기능 | 설명 |
|---|------|------|
| 1-1 | 실행 스크립트 더블클릭 실행 | `카카오톡전송기.command` 파일을 더블클릭하면 프로그램이 시작된다 |
| 1-2 | Python 설치 여부 확인 | Python3가 설치되어 있지 않으면 안내 메시지를 출력하고 종료한다 |
| 1-3 | 필요 패키지 자동 설치 | 최초 실행 시 pandas, pyperclip, openpyxl, flask, pyobjc 패키지를 자동으로 설치한다 |
| 1-4 | 웹 브라우저 자동 열기 | Flask 서버가 시작되면 웹 브라우저가 자동으로 열린다 (`http://localhost:5050`) |

## 2. 웹 인터페이스

| # | 기능 | 설명 |
|---|------|------|
| 2-1 | 등록형태 다중선택 | 등록형태(이월, 재등록, 신규)를 토글 버튼으로 다중 선택할 수 있다 (기본값: 전체 선택) |
| 2-2 | 연령대 다중선택 | 연령대(10대~60대 이상)를 토글 버튼으로 다중 선택할 수 있다 (기본값: 20대, 30대) |
| 2-3 | 엑셀 파일 선택 | 업로드 영역을 클릭하거나 드래그 앤 드롭하여 `.xlsx` 파일을 선택할 수 있다 |
| 2-4 | 드래그 앤 드롭 시각적 피드백 | 파일을 드래그할 때 업로드 영역이 강조 표시되어 드롭 가능 상태임을 알려준다 |
| 2-5 | 선택된 파일명 표시 | 파일 선택 후 선택된 파일명을 화면에 표시한다 |
| 2-6 | 발송 메시지 입력 | 텍스트 입력창에서 발송할 메시지를 직접 작성할 수 있다 (`{name}` 입력 시 회원 이름으로 자동 치환) |
| 2-7 | 전송 시작 버튼 | 파일을 선택한 후 "카카오톡 전송 시작" 버튼이 활성화된다 |
| 2-8 | 전송 중단 버튼 | 전송 중일 때 "전송 중단" 버튼(빨간색)이 표시되어 전송을 중단할 수 있다 |
| 2-9 | 필터/메시지 유효성 검사 | 등록형태, 연령대가 최소 1개 이상 선택되었는지, 메시지가 비어있지 않은지 확인한다 |
| 2-10 | 상태 표시 | 현재 상태(대기 중 / 전송 중)를 배지로 표시한다 |
| 2-11 | 실시간 로그 표시 | 전송 진행 상황을 실시간으로 로그 영역에 표시한다 (SSE 방식) |
| 2-12 | 로그 색상 구분 | 성공(초록), 실패(빨강), 경고/중단(노랑), 정보(파랑) 등 로그 유형별로 색상을 구분한다 |
| 2-13 | 완료/중단 알림 | 전송 완료 또는 중단 시 성공/실패 건수와 실패 대상자 목록을 알림(alert)으로 표시한다 |
| 2-14 | 버전 표시 | 화면 하단 푸터에 현재 버전(v1.0.0)을 표시한다 |

## 3. 엑셀 파일 처리

| # | 기능 | 설명 |
|---|------|------|
| 3-1 | 엑셀 파일 업로드 | 웹 브라우저에서 선택한 `.xlsx` 파일을 서버로 업로드한다 |
| 3-2 | 회원 목록 읽기 | 업로드된 엑셀 파일에서 회원 데이터(이름, 등록형태, 연령 등)를 읽어온다 |
| 3-3 | 타겟 멤버 필터링 | UI에서 선택한 등록형태와 연령대 조건에 해당하는 회원만 필터링한다 |
| 3-4 | 적용된 필터 로그 표시 | 현재 적용된 필터 조건(등록형태, 연령대)을 로그에 표시한다 |
| 3-5 | 전체/타겟 인원 수 표시 | 전체 회원 수와 필터링된 타겟 멤버 수를 로그에 표시한다 |

## 4. 카카오톡 제어

| # | 기능 | 설명 |
|---|------|------|
| 4-1 | 카카오톡 자동 실행 | AppleScript를 사용하여 카카오톡 앱을 활성화한다 |
| 4-2 | 카카오톡 창 감지 | macOS Quartz API로 카카오톡 메인 창을 감지한다 (창 ID 획득) |
| 4-3 | 카카오톡 준비 상태 확인 | 카카오톡 창이 열려 있는지 확인하고, 없으면 최대 3회 재시도한다 |
| 4-4 | 친구 목록 복귀 보장 | 검색 전 Esc + Cmd+1로 반드시 친구 목록 화면으로 이동한 뒤 검색을 시작한다 |
| 4-5 | 친구 검색 | 친구 목록에서 Cmd+F로 검색창을 열고 이름을 붙여넣기하여 친구를 검색한다 |
| 4-6 | 메뉴 클릭 방식 붙여넣기 | 클립보드에 텍스트를 복사한 후 메뉴바 > 편집 > 붙여넣기 방식으로 안정적으로 입력한다 |
| 4-7 | AX 친구 검증 | 접근성(AX) API로 검색 결과의 친구 이름(AXStaticText)을 정확한 문자열로 읽어 대상 이름과 일치하는지 검증한다. 검색창 값이 검색어와 일치할 때만 신뢰해 오발송을 방지한다 |
| 4-8 | 채팅방 열기 | 검색 결과에서 Enter 키로 채팅방을 연다 |
| 4-9 | 메시지 전송 | 사용자가 입력한 메시지에 이름을 치환하여 전송한다. 접근성(AX) API로 입력창에 직접 값을 넣어 Enter로 전송하며, 실패 시 클립보드 붙여넣기로 폴백한다 |
| 4-10 | 채팅방 닫기 | 메시지 전송 후 Esc 키로 채팅방을 닫는다 |
| 4-11 | 검색 초기화 | 다음 전송을 위해 Esc 3회로 모든 레이어를 닫고 Cmd+1로 친구 목록으로 돌아간다 |
| 4-12 | 랜덤 대기 시간 | 매크로 탐지 방지를 위해 각 단계마다 랜덤 대기 시간을 적용한다 (대상자 간: 1~3초, 검색 결과 로딩: 1~2초, 전송 후: 0.3~0.8초, 초기화 후: 0.2~0.5초) |

## 5. 전송 결과 처리

| # | 기능 | 설명 |
|---|------|------|
| 5-1 | 진행 상황 표시 | 각 대상자별로 `[현재/전체]` 형식으로 진행 현황을 로그에 표시한다 |
| 5-2 | 성공/실패 기록 | 각 대상자별 전송 성공 또는 실패 결과를 기록한다 |
| 5-3 | 실패 대상자 목록 | 전송 실패한 대상자의 이름 목록을 별도로 관리하고 표시한다 |
| 5-4 | 최종 결과 요약 | 전체 전송 완료 시 성공 건수/전체 건수와 실패 목록을 요약하여 표시한다 |
| 5-5 | 전송 중단 처리 | 사용자가 중단 버튼을 누르면 현재 대상자 처리 후 전송을 중단하고 결과를 표시한다 |

## 6. 오류 처리

| # | 기능 | 설명 |
|---|------|------|
| 6-1 | 전역 오류 핸들러 | Flask 서버에서 발생하는 예외를 전역적으로 처리하여 JSON 오류 응답을 반환한다 |
| 6-2 | 카카오톡 창 미감지 처리 | 카카오톡 창을 찾을 수 없으면 해당 대상자를 실패로 처리하고 다음으로 넘어간다 |
| 6-3 | AX 검증 실패 처리 | 친구를 찾을 수 없는 경우 해당 대상자를 건너뛰고 실패 목록에 추가한다. 실패 시에만 진단용 스크린샷을 1장 남긴다(화면 녹화 권한이 있을 때만) |
| 6-4 | 타겟 멤버 없음 처리 | 필터링 결과 타겟 멤버가 0명이면 경고 메시지를 표시하고 종료한다 |
| 6-5 | 중복 실행 방지 | 이미 전송이 진행 중인 경우 추가 실행을 방지한다 |

## 기술 스택

- **언어**: Python 3.9
- **웹 서버**: Flask
- **UI**: HTML/CSS/JavaScript (브라우저 기반)
- **엑셀 처리**: pandas, openpyxl
- **클립보드**: pyperclip
- **macOS 자동화**: AppleScript (osascript)
- **친구 검증/입력**: 접근성(AX) API (ApplicationServices / AXUIElement)
- **진단용 화면 캡처**: macOS `screencapture` (실패 시 1장, 선택)
- **실시간 통신**: Server-Sent Events (SSE)
